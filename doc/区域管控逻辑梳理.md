# 逻辑梳理
中间使用的ip地址均为模拟环境，可修改
区域管控逻辑基本与风淋门逻辑一致，对rcs来说还是风淋门。
区别是对接的ACS系统的报文不同

对应rcs上配置一个门编号使用q001 （默认，可自定义修改配置）
该门需要本地缓存记录状态


首先整体的设计思路是模拟一个门的状态，基本思路是rcs发开门请求，那么就转换成acs的请求进入区域的报文，返回我们需要的报文之后再更新开门状态，rcs自动获取到状态。



对rcs来说就是正常的发开门和查询即可，没有变化。

对于ACS的查询接口，可以定时轮询状态在web中显示，但实际似乎不需要调用，因为区域占用的情况下下发开门，那边有处理返回失败，我们这个转换只需要读到我们需要的报文后再更改门状态就可以。

## 区域管控逻辑

### 区域查询逻辑(单纯webui方便调试)

- 定时轮询查询，约5s一次（可配置）
- 请求进入区域前后及出区域前后均需要查询（确保信息实时性）后续请求逻辑自行添加，不额外添加

> curl --location --request GET 'http://10.68.2.10:5012/api/zones/status?zone_id=zone_001'

请求
```
{
  "permission_id": "perm_123456",//可配置,许可证
  "zone_id": "zone_001",//可配置
  "client_id": "iray",//可配置
  "status": "granted"//可配置
}
```
响应示例（已占用、不可用）
```
{
  "zone_id": "zone_001",
  "status": "occupied",//使用中的
  "occupied_by": "client_b"
}
```
响应示例（可用、可用）
```
{
  "zone_id": "zone_001",
  "status": "available"//可使用的
}
```



### 请求逻辑

#### 一. AGV rcs 请求开门q001

    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 1//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

#### 二、 ASAP 转换发送请求进入区域（post），根据api文档，一直尝试发送（间隔1s）

    > http://10.68.2.10:5012/sim/api/zones/enter
    {
    "zone_id": "zone_001",
    "client_id": "iray"
    }


        直到报文查询状态可用（"status": "granted"//的确）

        
    // **响应示例（成功）**
    // ```json
    // {
    //   "permission_id": "perm_123456",
    //   "zone_id": "zone_001",
    //   "client_id": "client_a",
    //   "status": "granted"//的确
    // }
    // ```


#### 三、 AGV 过门后rcs会下发控制关门报文

    rcs 会发送控制关门的请求，先将本地门状态缓存改成关门，然后转换成请求退出区域的报文，并一直尝试20次，间隔5s一次，并记录日志，直到"status": "released"停止重试。

    RCS发送关门请求
        > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 2//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

    ASAP发送退出区域请求

    curl --location --request POST 'http://10.68.2.10:5012/sim/api/zones/exit' \
--header 'Content-Type: application/json' \
--data-raw '{
  "zone_id": "zone_001",
  "client_id": "iray"
}'

    返回
    // {
    //     "zone_id": "zone_001",
    //     "client_id": "iray",
    //     "status": "released"
    // }

    返回"status": "released"表示成功







webui要求：区域管控需要显示一个门，和风淋门的显示一致，但是额外添加一个查询区域请求获取到的信息到卡片即可
