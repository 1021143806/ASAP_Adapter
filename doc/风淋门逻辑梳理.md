# 逻辑梳理
中间使用的ip地址均为模拟环境，可修改

风淋门对应rcs上配置的门编号使用1001和1002（默认，可自定义修改配置）

## 风淋门逻辑

1. AGV rcs请求开门

    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 1//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

2. ASAP 转换发送请求开门AB门和查询状态，根据api文档，一直发送

    > http://10.68.2.10:5012/sim/acs/door/DOOR01
    {
    "doorSerial": "DOOR01",//自动门序列号	与请求一致
    "command": "1",//"0"：无动作"1"：打开"2"：关闭
    "Direction": "1",//"1"：进"2"：出
    "RobotName": "AGV-RCS"
    }


    直到报文查询到开门到位
    {
        "doorSerial": "DOOR01",
        "doorStatus": "1",
        "command": "1",
        "code": "200"
    }

3. 主动上报rcs开门到位状态
    > http://10.68.2.10:7110/changeDoorStatus
    {
    "doorNum": "1001",      // 门编号
    "doorStatus": "1"       // 1=开门, 2=关门
    }

4. AGV 到位后rcs会下发控制关门报文，需要转换成对应的报文

    rcs 下发
    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 2//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

    > http://10.68.2.10:5012/sim/acs/door/DOOR01  
    {
    "doorSerial": "DOOR01",//自动门序列号	与请求一致
    "command": "2",//"0"：无动作"1"：打开"2"：关闭
    "Direction": "1",//"1"：进"2"：出
    "RobotName": "AGV-RCS"
    }

5. 持续获取进的门的状态，等待门关，并通知rcs关门

6. 持续获取另一个出的门的状态（反向同理）
    > http://10.68.2.10:5012/sim/acs/door/DOOR02
    等待返回开门到位
    {
        "doorSerial": "DOOR02",
        "doorStatus": "1",
        "command": "1",
        "code": "200"
    }

7. 主动上报rcs开门到位状态
    > http://10.68.2.10:7110/changeDoorStatus
    {
    "doorNum": "1002",      // 门编号
    "doorStatus": "1"       // 1=开门, 2=关门
    }

8. AGV 出门后rcs调用请求关门接口，转换成对应的请求关门

    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 2//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }


    > http://10.68.2.10:5012/sim/acs/door/DOOR02
    {
    "doorSerial": "DOOR02",//自动门序列号	与请求一致
    "command": "2",//"0"：无动作"1"：打开"2"：关闭
    "Direction": "2",//"1"：进"2"：出
    "RobotName": "AGV-RCS"//固定
    }
