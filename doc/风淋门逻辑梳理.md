# 逻辑梳理
中间使用的ip地址均为模拟环境，可修改

风淋门对应rcs上配置的门编号使用1001和1002（默认，可自定义修改配置）

## 问题
### 问题1 门同时开才能走问题
进入区域发生在只有一个门开的时候，此时车辆可能还在门外。应该等两个门都开后再进入区域？还是维持现状（一个门开就进入）？

等两个门都开后再进入，本质上模拟的就是一个门，但是用1s延迟来判断进出的方向

### 问题2 RCS 关门顺序 ⚠️

rcs的逻辑是这样的，如果门没关的话会一直发关门请求的报文，所以正确的逻辑是等待两个门都正在发送关门请求的时候才关门，然后并给ACS释放逻辑的报文。

### 问题3  单线程约束 

其实现在的逻辑与之前的完全控制不一样，现在本质倾向于单门，也就是两个门同时开和同时关，模拟的是一个很大很长的门，理论上支持多车通过的效率最优解，所以在给ACS发送预占区域的报文时，可以使用相同的设备号进行预占，只是说明RCS系统在用，但是具体什么车在用别管，用RCSAGV标识即可。

### 问题4 主动上报已移除

文档中多处写"主动上报rcs"，但我们刚移除了全部 report_door_open/close。RCS 通过轮询 /api/rcs/doorStatus 获取状态。

这个接口不需要了

## 风淋门逻辑

1. AGV rcs请求开门 （RCS端主动发送至该模块）

    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "1001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 1//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

2. ASAP 转换发送请求开门AB门和查询状态，根据api文档，一直发送

    > http://10.68.2.10:5012/sim/acs/door/DOOR01
    {
    "doorSerial": "DOOR01",//自动门序列号	与请求一致
    "command": "1",//"0"：无动作"1"：打开"2"：关闭
    "Direction": "1",//"1"：进"2"：出
    "RobotName": "AGV-RCS"
    }


    直到报文查询到开门到位
    {
        "doorSerial": "DOOR01",
        "doorStatus": "1",
        "command": "1",
        "code": "200"
    }


3. AGV 到位后rcs会下发控制关门报文，需要转换成对应的报文

    rcs 下发
    > http://10.68.2.10:5012/api/rcs/doorStatus
    {
        "deviceNum": "1746",//设备编号，标记用
        "doorCode": "q001",//门编号
        "payLoad": "0",//负载情况，没用
        "qrName": "81111096",//设备当前点位，没用
        "orderId": "",//任务id，没用
        "deviceCode": "test0",//设备序列号，设备唯一编号
        "status": 2//1请求开门，2请求关门
    }

    // 期望响应
    // {
    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }

    > http://10.68.2.10:5012/sim/acs/door/DOOR01  
    {
    "doorSerial": "DOOR01",//自动门序列号	与请求一致
    "command": "2",//"0"：无动作"1"：打开"2"：关闭
    "Direction": "1",//"1"：进"2"：出
    "RobotName": "AGV-RCS"//可使用rcs请求开门的设备号deviceNum
    }

4. 持续获取进的门的状态，等待门关，并通知rcs关门

5. 风淋过程，第三方控制，等待下一个门开

6. 持续获取另一个出的门的状态（反向同理）
    > http://10.68.2.10:5012/sim/acs/door/DOOR02
    等待返回开门到位
    {
        "doorSerial": "DOOR02",
        "doorStatus": "1",
        "command": "1",
        "code": "200"
    }


7. AGV 出门后rcs调用请求关门接口，转换成对应的请求关门

    //   "code": 1000,               // 1000=成功, 其他=失败
    //   "msg": "success",
    //   "data": null
    // }


    > http://10.68.2.10:5012/sim/acs/door/DOOR02

    "command": "2",//"0"：无动作"1"：打开"2"：关闭
